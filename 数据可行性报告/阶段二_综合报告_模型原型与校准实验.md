## Part 2：三模型原型说明

### Model A：Traditional Labor Market Benchmark

| 属性 | 说明 |
|------|------|
| **实现文件** | `models/model_a.py` |
| **Agent 结构** | 无个体 agent，使用人口份额（share_E, share_U, share_N） |
| **参数类型** | 同质代表性参数（均值） |
| **转移机制** | 确定性 Markov chain，转移概率由参数和环境决定 |
| **随机性** | 无（确定性模型） |
| **输出** | 月度 UR, LFPR, EPOP, 6 个转移率, hires/quits/layoffs |

**行为规则：**
- E→U: P(layoff) = layoff_rate_env / 100
- E→N: P(exit) = (1 - pp) × 0.06
- U→E: P(offer) × P(accept), 其中 P(accept) = Φ((mean_log_wage - log(rw)) / sd)
- U→N: P(discouraged) = (1 - pp) × 0.45
- N→U: P(re-enter) = pp × 0.04
- N→E: P(direct) = pp × 0.06 × P(offer) × P(accept) × 0.5

### Model B：Standard ABM（同质 Worker）

| 属性 | 说明 |
|------|------|
| **实现文件** | `models/model_b.py` |
| **Agent 结构** | N 个独立 Worker agent（默认 5000-10000） |
| **参数类型** | 所有 agent 共享相同参数（均值） |
| **转移机制** | 随机个体转移（与 Model A 相同的概率规则） |
| **随机性** | 有（个体层面的 Bernoulli 抽签） |
| **Worker 类** | `models/worker.py` — Worker.step() 方法 |

### Model C：Heterogeneous ABM（异质 Worker Types）

| 属性 | 说明 |
|------|------|
| **实现文件** | `models/model_c.py` |
| **Agent 结构** | N 个独立 Worker agent |
| **参数类型** | 异质——从分布中抽取，按就业状态区分 |
| **Worker Types** | 3 类（Active Searcher / Low-threshold / Marginal） |
| **分布** | reservation_wage ~ LogN(μ, σ); offer_arrival_belief ~ Beta(α, β); search_intensity ~ 3档离散 |
| **人口特征** | age_group × edu_group × sex → 12 组 participation_propensity |

### 三模型共同点与差异

| 维度 | Model A | Model B | Model C |
|------|---------|---------|---------|
| 时间步长 | 月度 | 月度 | 月度 |
| 环境路径 | 共享 | 共享 | 共享 |
| 状态空间 | E/U/N | E/U/N | E/U/N |
| 输出格式 | SimulationOutput | SimulationOutput | SimulationOutput |
| 行为规则 | 代表性均值 | 同质 Worker.step() | 异质 Worker.step() |
| 随机性 | 无 | 有 | 有 |
| worker heterogeneity | 无 | 无 | ✅ 有 |
| 参数搜索自由度 | 11 | 11 | 11 |

## Part 7：初步经济解释

### Model B 相对 Model A 的改进

Model B（Standard ABM）在 train 上相比 Model A 有 **88.6% 的总损失改善**（1,425 vs 12,469）。这是一个巨大的跳跃，主要因为：
- LFPR 从 10,793 降到 588（多 agent 随机性使 LFPR 动态更灵活）
- EPOP 从 897 降到 44
- 参数搜索找到了更匹配合成目标的参数组

**但 Model B 存在严重过拟合**（Overfit Ratio = 7.50），validation loss 是 train loss 的 7.5 倍。

### Model C 相对 Model B 的增益

**核心发现：Model C 的优势不在 Train，而在 Validation。**

| 维度 | Model B | Model C | C 的优势 |
|------|---------|---------|---------|
| Train Loss | **1,425** | 1,559 | B 略优（+9.4%）|
| **Valid Loss** | 11,008 | **1,876** | **C 碾压 (-83.0%)** |
| **Overfit Ratio** | 7.50 | **1.20** | **C 几乎无过拟合** |
| Train 赢的目标数 | 5 | **7** | C 赢更多 |

**经济解释：异质性的核心贡献是泛化能力，而非训练集拟合。**

同质模型可以通过精调单一参数组来紧密拟合训练期的平均行为，但当经济环境变化（如从 COVID 恢复期进入正常期），它无法自适应——因为它没有组合效应。

异质模型拥有三个同质模型不具备的结构性优势：

1. **匹配市场竞争**——有限岗位按搜索强度优先分配，产生非线性聚合效应
2. **类型特定行为规则**——Active Searcher 的裁员风险更低（×0.85），Marginal Worker 退出更快（×1.3）
3. **组合效应（composition effect）**——当经济条件变化时，不同类型 worker 的响应速度不同，改变总体中各类型的混合比例

### C 优于 B 的 7 个指标

1. **unemployment_rate（-97.4%）**——匹配市场使 UR 对 vacancy 变化更敏感
2. **flow_UE（-23.7%）**——declining reservation wage + 匹配优先级使失业者逐步接受 offer
3. **flow_EU（-2.7%）**——类型特定裁员概率更真实
4. **flow_EN（-7.5%）**——退出规则修复 + 类型乘数使退出率更合理
5. **flow_NU（-83.8%）**——Marginal Worker 的低 re-entry 概率更真实
6. **hires_rate（-6.1%）**——匹配市场使招聘率更合理
7. **quits_rate（-17.4%）**——只有异质模型才能产生有意义的自愿离职率变化

### C 劣于 B 的 4 个指标

1. **lfpr（+54.0%）**——pp_delta=+0.04 使整体 LFPR 偏高
2. **flow_NE（+8.7%）**——N→E 直接入职率略高
3. **flow_UN（+5.5%）**——U→N 退出率略高
4. **layoffs_rate（+20.2%）**——类型乘数使 layoff 分布偏离均值

### 过拟合分析

| 指标 | Model B | Model C |
|------|---------|---------|
| Overfit Ratio (Valid/Train) | **7.50** ❌ | **1.20** ✅ |
| Train Std (across seeds) | 256 | 78 |
| Valid Std (across seeds) | 623 | 323 |

**Model C 的过拟合比率仅 1.20——这是搜索匹配类模型中罕见的优秀泛化性。** 原因：
1. 异质性参数创造了"结构性正则化"——模型无法通过单一参数组精确拟合训练期，但这恰好使它对新数据更鲁棒
2. 匹配市场的竞争机制产生了内生的均衡调节——当某类 worker 太多/太少时，匹配概率自动调整
3. 3000 个 agent 的大群体效应平滑了个体噪声
